Variable Manipulation
(8分)

CYLL has an integer-typed variable XXX whose initial value is 1. The variable can be updated in each step by applying either of the two operations:

  1. Multiply the variable by a fixed integer KKK: X=X×KX = X \times KX=X×K

  2. Add an integer TTT (1≤T≤101 \le T \le 101≤T≤10) to the number: X=X+TX = X + TX=X+T

Given two integers NNN and KKK as inputs, can you help CYLL to decide the minimum number of steps required before she obtains the number NNN as the variable value?

Format of functions：

int FindMinSteps(int N, int K);

Here NNN (1≤N≤106 1\le N \le 10^6 1≤N≤10​6​​) is the target number and KKK (1≤K≤N 1 \le K \le N 1≤K≤N) is the multiplication factor, which are integers as described in the problem specification.

The function is expected to return the minimum number of steps before the variable XXX reaches value NNN. (See the sample input/output for a concrete example.)

Sample program of judge：

#include <stdio.h>

int FindMinSteps(int N, int K);

int main()
{
  int N, K;

    scanf("%d%d", &N, &K);

    printf("%d", FindMinSteps(N, K));

    return 0;
}    

/* Implement the 'FindMinSteps' function here */

Sample input：

101 2

Sample output：

6

Sample explanation :

step 1: X=1+10=11 X = 1 + 10 = 11 X=1+10=11

step 2: X=11∗2=22 X = 11 * 2 = 22 X=11∗2=22

step 3: X=22∗2=44 X = 22 * 2 = 44 X=22∗2=44

step 4: X=44+6=50 X = 44 + 6 = 50 X=44+6=50

step 5: X=50∗2=100 X = 50 * 2 = 100 X=50∗2=100

step 6: X=100+1=101 X = 100 + 1 = 101 X=100+1=101

Note that this is not the only way to reach 101 in 6 steps, however, we care about the minumum number of steps, which is unique, rather than how you reach there.
